# やねうら王との探索機能比較

変成将棋 AI の現実装と、やねうら王（Stockfish ベース）の機能を比較し、今後の実装方針を整理する。

---

## 実装済み

| 機能 | 概要 | 備考 |
|---|---|---|
| αβ探索 + 反復深化 | 基本探索フレームワーク | |
| 置換表（TT） | 2^20エントリ ≒ 14MB、完全/下限/上限フラグ | TT ヒット率 10〜35%（やねうら王は40〜60%） |
| LMR（Late Move Reduction） | 3手目以降の静かな手を depth-2 で探索 | 残深さ >= 4 で発動。深さ4では効果薄 |
| NMP（Null Move Pruning） | 残深さ >= 3 かつ staticEval >= β でパス手探索 | R=3 固定 |
| 静止探索（Quiesce Search） | 駒取り手のみ継続して局面を安定化 | 深さ10手で打ち切り |
| Futility Pruning | 浅いノードで staticEval + margin < α なら静かな手をスキップ | 残深さ1=飛車1枚、残深さ2=飛車2枚 |
| Delta Pruning | Quiesce で取る駒 + stand_pat < α なら打ち切り | |
| History Heuristic | βカットした静かな手の深さ二乗を蓄積 | オーバーフロー時に全体を半減 |
| Killer Move | 深さごとに最大2手保持 | |
| TT 最善手優先 | 置換表から得た最善手を手順先頭に並べる | |
| 駒取り優先 | 取る駒の価値でスコアリング（近似 MVV） | LVA（動かす駒価値）は未考慮 |
| NNUE 評価（float32） | HalfKP 特徴量 + 変成将棋専用バケット | フォールバック用 |
| NNUE 評価（INT8） | INT16 加算器 + AVX2 pmaddubsw | 優先使用。SIMD 最適化済み |
| NNUE インクリメンタル更新 | 差分更新（diff）、玉移動時はフル再計算（scratch） | diff 94%、scratch 6% 程度 |
| Lazy Update | SetDirty → Eval 時に Refresh するまで加算器更新を遅延 | TT ヒット時のみ効果 |
| 定跡（Opening Book） | SFEN ハッシュベースの定跡テーブル | |

## 未実装（やねうら王にあるもの）

| 優先度 | 機能 | 概要 | 期待効果 |
|---|---|---|---|
| ★★★ | **千日手検出** | 同一局面 4回繰り返しで引き分け判定 | 終盤ループ解消。TT ヒット率100%ループを断ち切る |
| ★★★ | **Aspiration Window** | 反復深化の各深さで前回スコア ± マージンのウィンドウから探索開始 | TT ヒット率向上、探索ノード大幅削減 |
| ★★☆ | **Check Extension** | 王手局面で探索深さを +1 延長 | 詰み読みの精度向上、棋力直結 |
| ★★☆ | **MVV-LVA** | 駒取りスコア = 取る駒価値 - 動かす駒価値 / 定数 | 手順並べ替えの精度向上 |
| ★★☆ | **SEE（Static Exchange Evaluation）** | 駒交換の損得を静的評価。Quiesce のフィルタリングに使用 | 変成将棋では損する交換が攻めに使われるため慎重に調整が必要 |
| ★☆☆ | **Countermove Heuristic** | 直前の手に対する反論手を優先 | History の補完 |
| ★☆☆ | **Singular Extension** | 置換表の最善手が他手より明らかに優れている場合に延長 | 実装複雑、効果大 |
| ★☆☆ | **IID（Internal Iterative Deepening）** | TT 最善手がない深いノードで浅い探索を先行 | TT ヒット率補完 |
| ★☆☆ | **Lazy SMP** | マルチスレッド並列探索（置換表共有） | コア数倍の速度、最後に実装 |

## 速度比較（深さ4・300手通し）

| 構成 | 速度 |
|---|---|
| αβ + TT + LMR + NMP のみ | 918 ms/手 |
| + Quiesce + History + Killer + Futility | 505 ms/手 |
| + Quiesce 深さ10打ち切り | 438 ms/手 |

---

## 備考

- NNUE の SIMD 実装（AVX2）は C++ のやねうら王と同等レベルで、言語差による速度差は小さい
- 速度差の主因は探索アルゴリズムの質（TT ヒット率・枝刈り効率）
- 変成将棋固有の事情：千日手が深さ4では225手目頃から発生するため、千日手検出が急務
