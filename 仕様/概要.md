# 変成将棋 プロジェクト概要

## 何を作っているか

「変成将棋」は **玉将も成れる** 変則将棋である。
玉が成ると「獅王」になり、チェスの獅子に近い特殊な動きをする。
これを遊べる WPF アプリと、自己対局で学習する AlphaZero 型 AI を合わせて開発している。

---

## プロジェクト構成（一覧）

```
変成将棋.Engine/          ← ゲームロジック（C# クラスライブラリ）
変成将棋.Engine.Native/   ← Python 学習用 NativeAOT DLL
変成将棋.Engine.Tests/    ← xUnit テストプロジェクト
変成将棋.AI/              ← 対局 AI（ONNX + MCTS）
変成将棋/                 ← WPF デスクトップアプリ
ml/                       ← Python 学習スクリプト群
仕様/                     ← 設計ドキュメント
```

## レイヤー別の役割

### (1) ゲームエンジン（`変成将棋.Engine`）

将棋ルールを実装する純 C# ライブラリ。WPF・AI・Python いずれからも参照される。

**主要クラス：**

| クラス/構造体 | 役割 |
|---|---|
| `C盤面` | 局面の状態保持。`Apply`/`Undo` で手を適用・取消し |
| `C合法手生成器` | 擬似合法手生成 + 王手放置フィルタ |
| `C攻撃テーブル` | マジックビットボード（スライド駒の攻撃テーブル） |
| `S利きビット` | 81升を 2 つの ulong で表すビットボード |
| `S手` | 指し手（4バイト値型、stackalloc で扱う） |
| `S升座標` | 升目座標（1バイト値型、上位4bit=段・下位4bit=列） |
| `C利き管理` | 全利きをゼロから再計算（テスト・検証用） |
| `C将棋FE表記` | SFEN 文字列のパース・シリアライズ |

**王手放置の判定（`Is王手放置`）：**

マジックビットボードによる逆引き方式。
「玉位置から各駒種のテーブルを引き、相手の駒種別ビットボードと AND」で O(駒種数 ≈ 10) で判定する。
差分更新方式（旧実装）の XOR-AND バグを根絶するために置き換えた。

### (2) WPF アプリ（`変成将棋`）

盤面表示・駒の移動操作・対局管理を行う GUI。
エンジンの `C盤面` を直接参照し、AI プレイヤー（`変成将棋.AI`）と組み合わせて動く。

### (3) 対局 AI（`変成将棋.AI`）

| クラス | 内容 |
|---|---|
| `CONNXPlayer` | ONNX モデルのグリーディ方策（推論のみ） |
| `CMCTSPlayer` | MCTS + ONNX（C# ネイティブ、橋のオーバーヘッドなし） |

### (4) Python 学習系（`ml/`）

AlphaZero 方式の自己対局学習。GPU 環境（Colab T4）と CPU ローカルの両方で動く。

| ファイル | 役割 |
|---|---|
| `network.py` | ResNet 10ブロック/128ch（policy + value） |
| `train.py` | 学習ループ（Gumbel AlphaZero + PER） |
| `self_play.py` | 自己対局・優先度付き経験再生バッファ |
| `batched_mcts_native.py` | バッチ推論対応 MCTS（NativeAOT 版） |
| `native_game_env.py` | NativeAOT DLL を ctypes で呼ぶラッパー |
| `train_local.py` | CPU ローカル用の設定済み起動スクリプト |
| `colab_train.ipynb` | Colab 用ノートブック（Drive から継続学習） |

### (5) NativeAOT DLL（`変成将棋.Engine.Native`）

Python の ctypes から直接呼ぶための C スタイル API。
Python.NET（~8ms/call）の代わりに使用することで ~0.1ms/call に短縮。
Windows は `.dll`、Linux（Colab）は `.so` としてビルドする。

## ゲームのルール概要

標準将棋との主な差異：

| 元の駒 | 標準将棋の成駒 | 変成将棋の成駒 | 差異 |
|---|---|---|---|
| 歩兵 | と金 | と金 | 同じ |
| 香車 | 成香（金と同じ動き） | **竪行**（縦スライド＋横1升） | 名前も動きも変わる |
| 桂馬 | 成桂（金と同じ動き） | **騎兵**（チェスナイト、8方向跳び） | 名前も動きも変わる |
| 銀将 | 成銀（金と同じ動き） | **麒麟**（斜め1升＋縦横2升跳び） | 名前も動きも変わる |
| 金将 | 成れない | **鳳凰**（縦横1升＋斜め2升跳び） | 金将が成れる（新規） |
| 角行 | 龍馬 | 龍馬 | 同じ |
| 飛車 | 龍王 | 龍王 | 同じ |
| 玉将 | 成れない | **獅王**（距離2以内すべて、1手2回移動可） | 玉将が成れる（新規・条件あり） |

獅王（玉の成駒）は1手で2回移動でき、途中で駒を取れる。
ただし1回目に戻る（パス）こともできる。
玉将が獅王に成るには「自玉が敵陣3段目以内、かつ相手の玉将（未成）が自陣3段目以内」という追加条件がある。

## データフロー

```
【学習時】
Python MCTS
  └─ ctypes ─→ shogi_engine.so（NativeAOT）
                  └─ 変成将棋.Engine（合法手生成・局面適用）
  └─ PyTorch ─→ GPU NN 推論
  └─ 学習結果 ─→ checkpoints/model_final.pt（Google Drive 経由で Colab ↔ ローカル共有）

【対局時】
WPF UI ─→ CMCTSPlayer（C#）
              ├─ 変成将棋.Engine（C# ネイティブ直結）
              └─ OnnxRuntime ─→ model.onnx
```
